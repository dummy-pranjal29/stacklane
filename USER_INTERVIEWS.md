## Interview 1 — Sarah Chen, CTO at 12-person AI startup

**Background**: Sarah runs engineering at a Series A startup. Team of 12, 6 engineers.

**Direct quotes**:

- "We're paying for Cursor, GitHub Copilot, and ChatGPT Enterprise. Honestly, I have no idea if we're overspending."
- "The biggest pain is that nobody owns the bill. It gets charged and we just... pay it."
- "If you told me we could save 30%, I'd probably try it. But I don't have time to evaluate every tool myself."

**Most surprising**: She didn't know Copilot had team pricing; thought it was $10/month per person. They're likely overpaying by 50%.

**Design impact**: Emphasized that founders need a "second opinion" more than detailed explanations. Changed our CTA from "See all recommendations" to "Is there a cheaper way?"

## Interview 2 — Mark T., Indie Hacker & Solo Founder

**Background**: Mark runs a portfolio of three micro-SaaS apps. He operates solo but works with 2 contractors.

**Direct quotes**:

- "I have subscriptions to Claude Pro, ChatGPT Plus, and Midjourney. I only really use Claude for coding."
- "The $20 here and there adds up. It feels like death by a thousand subscriptions."
- "I cancel and resubscribe constantly based on which model is winning the benchmark that month."

**Most surprising**: He prefers paying retail for standard UI tools rather than using cheaper API keys because he values the native chat interfaces over raw savings.

**Design impact**: Realized that API credit recommendations are only useful for enterprise/teams. Solo founders won't trade the ChatGPT/Claude web UI for a 40% discount on API credits. Added a "team size" conditional before recommending API transitions.

## Interview 3 — David L., VP Engineering at Series B SaaS

**Background**: David manages a team of 45 engineers.

**Direct quotes**:

- "Finance keeps asking me to justify the $1,500/mo we spend on OpenAI."
- "We gave everyone Cursor licenses, but half the team is still using IntelliJ without any AI."
- "If I had a dashboard showing who actually uses the credits, I could cut the bill in half today."

**Most surprising**: The actual cost isn't the problem for him; it's the lack of visibility into utilization. Finance is pressing him, not his own desire to save money.

**Design impact**: Added "Benchmark Mode" to our roadmap. For now, ensured the PDF report is heavily tailored so an Engineering Manager can hand it directly to a CFO as a "cost optimization initiative."
